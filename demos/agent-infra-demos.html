Demo 01: 沙箱层 — AgentBay 云端沙箱

沙箱层 / Sandbox

什么是沙箱？

沙箱 = Agent 执行工具调用的隔离环境。LLM 生成的代码完全不可信，必须在隔离的 VM/容器中执行。

本 Demo 在 AgentBay（阿里云无影）上创建了一台完整的云电脑 VM（4核/7GB/99GB），演示了从创建到销毁的完整生命周期。

Demo 02: 编排层 — ReAct 编排循环

编排层 / Orchestration

什么是编排层？

编排层 = Agent 的"大脑"，决定做什么、用什么工具、按什么顺序。

核心模式是 ReAct 循环：Think → Tool Call → Observe → Repeat。本 Demo 用 MockLLM 演示编排结构（因环境内 LLM API 不可用）。

$ python3 demos/02_orchestration/demo_orchestration_loop.py
====================================================================== Demo 02: Agent 编排层 — ReAct 编排循环 对应层: 编排层 / Orchestration ====================================================================== ──────────────────────────────────────────────────────────── 任务 1/4 [USER] 计算 15 * 37 + 12 等于多少 [THINK] 步骤 1: 需要调用工具 'calculator' [TOOL_CALL] calculator({'expression': '15 * 37 + 12'}) [OBSERVE] 结果 (0ms): 567 [ANSWER] 基于工具调用结果，任务完成。结果: 567 ──────────────────────────────────────────────────────────── 任务 3/4 [USER] 搜索北京今天天气怎么样 [THINK] 步骤 1: 需要调用工具 'web_search' [TOOL_CALL] web_search({'query': '搜索北京今天天气怎么样'}) [OBSERVE] 结果: 北京今天晴，气温 28°C，湿度 45% [ANSWER] 基于工具调用结果，任务完成。 ──────────────────────────────────────────────────────────── 任务 4/4 [USER] 读取 /etc/hostname 文件的内容 [THINK] 步骤 1: 需要调用工具 'read_file' [TOOL_CALL] read_file({'path': '/etc/hostname'}) [OBSERVE] 结果: sessionf2ec2429c2f74ad0ac-zz2pk [ANSWER] 任务完成。

Demo 03: MCP 协议 — Server + Client 交互

协议层 / Protocols

什么是 MCP？

MCP (Model Context Protocol) 是 Anthropic 发布的开放协议，定位：Agent ↔ 工具的标准化接口。

类比 USB 接口 — 让任何 Agent 接入任何工具。已被 AWS GA 采用，有数千个 MCP Server 生态。

$ python3 demos/03_mcp_protocol/mcp_client.py

可观测性 / Observability

为什么需要可观测性？

Agent 是多步推理系统 — 每次 LLM 调用、每次工具调用都是黑盒。可观测性让你看到：调用链、延迟、Token 用量、错误点。

产品架构拆解 — Codex vs Claude Code vs Devin

架构对比

同样的三层，不同的组合方式

编排、运行时、沙箱是三个独立的层，但不同产品的组合方式完全不同。

维度OpenAI CodexClaude CodeDevin ($4.5B)
编排层OpenAI Agents SDKClaude 模型推理 + 工具框架自研多 Agent 编排引擎
运行时Ona (Gitpod) VM本地 CLI 进程 (Node.js)自建云 VM
沙箱= Runtime (合并)无独立沙箱 / E2B= Runtime (合并)
Runtime=Sandbox?是 (Agent 跑在沙箱内)否 (分离或无沙箱)是 (全自建)
持久化✅ (收购 Ona 解决)❌ (依赖本地)✅
Browser 内置❌❌✅
成本高 (收购)低 (无沙箱)最高 (全自建)
Codex 架构 — Runtime 与 Sandbox 合并： ┌── ① 编排层 ──────────────────┐ │ OpenAI Agents SDK │ └──────────┬───────────────────┘ ↓ ┌── ② Runtime + ③ Sandbox ────┐ │ Ona (Gitpod) 持久化 VM │ │ Agent 进程 + 代码执行 │ │ 都在同一台 VM 里 │ └──────────────────────────────┘
Claude Code 架构 — Runtime 与 Sandbox 分离： ┌── ① 编排层 ──────────────────┐ │ Claude 模型推理 │ └──────────┬───────────────────┘ ↓ ┌── ② Runtime ─────────────────┐ │ CLI 进程 (你的电脑 / 云端) │ └──────────┬───────────────────┘ ↓ (可选) ┌── ③ Sandbox ─────────────────┐ │ 无 / E2B / OpenSandbox │ │ 用权限控制代替隔离 │ └──────────────────────────────┘
Devin 架构 — 三层全自建： ┌── ① 编排层 ──────────────────┐ │ 自研多 Agent 引擎 │ └──────────┬───────────────────┘ ↓ ┌── ② Runtime + ③ Sandbox ────┐ │ 每任务一台独立 VM │ │ ├── Shell (命令行) │ │ ├── Browser (Chromium) │ │ └── Editor (VS Code 风格) │ │ 可并行多 VM │ └──────────────────────────────┘
